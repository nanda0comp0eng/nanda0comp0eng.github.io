The Basics of HTTP and HTTPS

The internet relies on conversations between devices called clients and servers. These conversations happen using HTTP, a language specifically designed for the web. When you type a web address (like medium.com) into your browser, it acts as a client asking a server for information. The server retrieves the requested data (often web pages) and sends it back.

The Problem with HTTP: Insecurity

Imagine sending a postcard with your login details written on it! That's kind of what happens with regular HTTP, anyone can snoop on the information being sent back and forth. This is a major security risk, especially on public Wi-Fi networks.

HTTPS is like sending your postcard in a sealed envelope. Even if someone intercepts it, they can't see what's inside because it's encrypted. This is why HTTPS is becoming the standard for websites, and most browsers will soon block insecure HTTP websites altogether.

HTTP Methods and Response Status Codes

The web relies on a special language called HTTP for communication. Imagine your browser and the website you visit are having a conversation. This conversation uses different methods (like requests and responses) and secret codes (like status codes) to understand each other.

Methods: Telling the Website What to Do

  • GET: This is like asking the website to "show me something."
  • POST: This is like giving something to the website.
  • HEAD: This is like asking the website "what do you have?" without actually downloading anything.
  • PUT & DELETE: These are more advanced methods used to create or remove things on the website.
  • OPTIONS: This is like asking the website "what can you do?" It tells you what other methods the website understands.

Codes: Understanding the Website's Response

The website responds to your requests using codes. These codes are like secret messages that tell you if everything went well or if there's a problem.

Information Responses (100s)

  • 100 Continue
  • 101 Switching Protocols
  • 102 Processing (WebDAV)
  • 103 Early Hints

Successful Responses (200s)

  • 200 OK
  • 201 Created
  • 202 Accepted
  • 204 No Content
  • 206 Partial Content

Redirection Messages (300s)

  • 301 Moved Permanently
  • 302 Found
  • 303 See Other
  • 304 Not Modified
  • 307 Temporary Redirect

Client Error Responses (400s)

  • 400 Bad Request
  • 401 Unauthorized
  • 403 Forbidden
  • 404 Not Found
  • 405 Method Not Allowed
  • 410 Gone

Server Error Responses (500s)

  • 500 Internal Server Error
  • 502 Bad Gateway
  • 503 Service Unavailable
  • 504 Gateway Timeout
